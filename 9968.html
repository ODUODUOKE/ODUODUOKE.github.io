Hexo博客文件迁移新环境部署

发表于2023-06-13|更新于2025-12-11
|总字数:356|阅读时长:1分钟|浏览量:

文档位于云仓库

Hexo文档博客已上传gitee仓库

新环境部署

新环境重新部署/迁移注意事项

  1. 确保电脑已安装nodeJs和Git
  2. npm install hexo-cli -g #全局安装hexo-cli客户端
  3. npm install #项目局部安装依赖
  4. hexo g #打包 (win10/11的powershell无法执行hexo命令，用cmd命令提示符即可)
  5. hexo s #运行 (同上所述)

一键运行脚本

一键运行脚本文件目录[ other ]下

  1. 一键创建页面hexo.bat
  2. 一键部署启动hexo.bat
  3. 一键部署GitHub_Pages.bat

注意：若hexo文档目录与脚本.bat不符合,请修改bat文件中最新目录位置，否则脚本启动有问题！

注意事项

  1. 创建发布目录,用于hexo g打包页面资源文件发布提交github
  1. 以克隆方式拉取github上仓库的项目到本地，主要是获取.git配置

  1. 将del.bat脚本复制到.deploy目录

del.bat 脚本会在提交推送文件后清空当前目录文件

其他补充

自定义css样式

主题theme目录下的css下的custom目录未自建目录,非主题初始化自带。主要作用是对博客主题自定义美化的css样式配置等；

自定义资源文件

可配置相关资源文件图片等，通过命令hexo -g打包生成html静态页面会附带当前资源等。
